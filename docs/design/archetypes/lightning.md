# 闪电流派：导电网络

> 本文是玩法/美术规格草案，不是实现说明。文中全部数值均为待试玩锚点，须以实际对局反馈校准。

## 定位与幻想

闪电把分散命中变成敌群之间持续自发的电网。玩家先让多个敌人带电，再利用聚团、路径与位置关系让电弧持续跳接；当网络足够密集时，雷暴将整张网瞬间放大为全场落雷。

## 印象与关键词

极化、连接、邻近、跳链、聚团、脉冲、过载、落雷。闪电应带来清晰的“谁和谁连上了”的空间推理，而不只是随机跳伤。

## 核心循环

闪电链命中多个目标；极化使被命中的敌人带电；相邻带电敌人之间持续放出电弧；玩家借由敌人聚集或位移维持更多连接；当带电目标数量和距离同时满足条件，雷暴发出一次全场链式落雷。收益来自网络密度而非单个目标上的叠层。

## 弹珠与遗物协同

| 角色 | 名称 / id | 玩家可感知作用 |
|---|---|---|
| 起点 | 闪电链 / `lightning` | 命中后向其他目标跳链，铺开带电候选。 |
| 网络 | 极化 / `polarize` | 链命中的敌人带电，靠近时自动互放电弧。 |
| 兑现 | 雷暴 / `thunderstorm` | 网络达到密度阈值后触发全场链式落雷。 |
| 桥梁 | 带电球 / `live_wire` | 共振球反弹为最近敌人提供放电，连接回响。 |

雏形由闪电链和极化构成，必须先让玩家感到“聚在一起就会自己放电”；雷暴是网络成形后的高潮，而非每隔数秒自动释放。冰与炸弹带来的聚团和位置重排，应自然提高闪电的网络质量。

## 视觉语言

主色是蓝白电芯、靛蓝外晕和少量紫色高压边缘。带电敌人身上有稳定但不刺眼的环形电荷标记；真正连通时，敌人间绘出短促、分段、有方向感的电弧。雷暴不应是一片蓝白噪光，而应从高处依次落下数道粗细不同的雷柱，照亮已连接的网络。

## 图标方向

闪电链为折线电弧穿过三枚圆点；极化为一枚敌影外包正负电荷环；雷暴为云下三道分叉落雷；带电球为圆球绕着一圈小电弧。图标线条可尖锐，但须保留明显的连接对象关系，避免只画单道闪电。

## 特效需求

- 闪电链每次跳转有明确的起点、终点和极短停顿，帮助数清目标。
- 带电状态是稳定标记，网络电弧则是按节拍闪现，二者不可混为同一亮度。
- 连线距离接近断开时，电弧变细、闪烁加快，为玩家提供位置反馈。
- 雷暴触发前先显示全网同步脉冲，随后落雷按可读次序传导，不应瞬间淹没画面。
- 声音从细小电流、连接嗡鸣到低沉雷击分层，避免全程高频噪声。

## 溅射物/区域/实体规则

带电是附着在敌人身上的短寿命状态；它不产生固定地面区域。两个带电敌人进入待试玩半径 80 内时建立一条可见电弧，电弧按固定节拍造成伤害；同一对敌人有独立冷却，不能在完全重叠时无限高频结算。每个带电敌人可同时连接的邻居数要有上限，首轮建议从 2–3 条连接试起，并优先选择最近目标。

雷暴只在带电敌人达到待试玩数量阈值 4/3/3 且形成近距离网络时触发。一次雷暴按可读顺序选取网络成员进行链式落雷；每个目标在同一次雷暴中最多被主雷命中一次。落雷可刷新或补充带电，但必须有明确的本次触发上限，不能让落雷立刻无穷重开雷暴。

## 反馈与可读性

玩家应该能在不读面板的情况下回答三件事：哪些敌人带电、哪些敌人已相连、距离雷暴还差什么。带电标记、电弧线和雷暴阈值脉冲分别承担这三层信息。大量敌人聚集时，电弧只保留最强或最近的若干条，其余以小型端点闪光替代，防止形成不可辨识的蓝色网团。

## 平衡边界

闪电需要多个带电敌人与足够近的距离，不能在单体或分散敌群中仍保持同等输出。电弧伤害暂以 2/3/5 为锚点，连接半径暂以 80 为锚点；实际强度要按平均连接数评估。雷暴应是成型奖励而非常驻循环，触发后需要短暂的网络重建期。带电持续时间、每点连接数和每对冷却是主要限制阀，不能同时放宽。

## 与其他流派的桥梁

带电球 `live_wire` 使回响的共振反弹向最近敌人放电，并可协助建立极化网络。冰球与爆破击飞都能把敌人推向更适合导电的密度，这是明确的涌现协同，不增加额外桥梁遗物。火与毒可以与闪电共存，但不应为覆盖所有组合而强行增加元素反应。

## 待确认问题

1. 网络连接上限应以每个敌人为单位，还是以全场总电弧数为单位，哪种更可读？
2. 雷暴的阈值更应看带电敌人数量、连接数量，还是两者共同满足？
3. 带电持续时间多长才能鼓励聚团而不让网络轻易常驻？
4. 电弧在密集敌群中的简化规则怎样设计，才能保留网络感而不遮挡战场？
