# 冰封流派：物理改造战场

> 本文是玩法/美术规格草案，不是实现说明。文中全部数值均为待试玩锚点，须以实际对局反馈校准。

## 定位与幻想

冰封将敌人从“被攻击者”转化为可摆布的台面棋子。冻结后的敌人是可推动的冰球、暂时的障碍与可被击碎的风险源；玩家用弹珠轨迹和敌群站位制造碰撞、堵路、碎裂与碎片齐射。

## 印象与关键词

冷硬、停滞、折射、棋子、撞击、碎裂、临时墙、滑行。冰应让台面变得更具空间感，而非只是给敌人挂一层蓝色减速。

## 核心循环

冰霜弹珠累积霜冻并冻结目标；永冻将其保留为更大、更稳定的冰球障碍；玩家把弹珠或其他被位移的敌人撞向冰球，改变战场线路；冰爆解除一个冻结目标，把已积累的霜冻转换为定向冰碎片，完成一次有代价的破局。

## 弹珠与遗物协同

| 角色 | 名称 / id | 玩家可感知作用 |
|---|---|---|
| 起点 | 冰霜弹珠 / `blue_marble` | 累积霜冻并使敌人冻结。 |
| 触发 | 碎冰锤 / `ice_hammer` | 命中冻结敌人时造成局部碎裂。 |
| 台面 | 永冻 / `permafrost` | 冻结敌人化为更大的可推动冰球障碍。 |
| 兑现 | 冰爆 / `cryoclasm` | 消耗冻结与霜冻，射出冰碎片抛射物。 |
| 桥梁 | 黑冰 / `black_ice`、冻脆 / `brittle` | 分别连接回响与炸弹。 |

雏形是冰霜弹珠加永冻：先证明“做棋子”有乐趣；冰爆提供主动拆棋与远程收束；碎冰锤让已有冻结目标在近距离碰撞时更有回报。冰不能依靠持续伤害成为纯粹的安全流。

## 视觉语言

主色为深靛蓝、冰青、白色折光与极少量灰蓝阴影。冻结不是简单的半透明滤镜：敌人内部应有放射状裂纹和被封住的动作残像；永冻后的冰球体积更大、轮廓更圆，边缘带霜刺。冰碎片呈锐利三角形折射，黑冰则是接近墨色、带蓝紫反光的薄滑面。

## 图标方向

冰霜弹珠为内部有雪花裂纹的蓝白圆珠；碎冰锤为敲击冰块的锤面；永冻为包住敌人轮廓的巨大冰球；冰爆为向外放射的六枚冰片；黑冰为黑蓝镜面上的反光弧线；冻脆为覆盖霜纹的裂壳炸弹。图标可使用尖锐晶体轮廓，但保持中心物体清晰。

## 特效需求

- 霜冻积累以逐层加密的晶霜环表现，冻结瞬间有一次短而干净的结晶收束。
- 冰球被推动时显示低速滑痕、重量感和碰撞前的影子偏移。
- 碰撞优先呈现接触点的白亮裂纹，再呈现扩散碎屑，避免大面积蓝雾。
- 冰爆先让冻结外壳碎开，再射出可追踪的冰片；目标仍存活这一信息必须清楚。
- 黑冰反弹应带细长冷色擦痕和不同于普通墙面的滑音。

## 溅射物/区域/实体规则

冻结敌人保持为场上可见实体：基础冻结可恢复行动；拥有永冻时，冻结体积放大且至少维持到本发结束，成为可推动的冰球障碍。冰球与敌人或墙面碰撞时产生可见的冲击与碎裂伤害；冰球不应无限加速，也不能在一个区域反复无代价来回结算。

冰爆只能选择一个已冻结目标，解除其冻结并清空其霜冻，但该目标仍然存活。随后向周围或前方发射 3/4/6 枚冰碎片，数值为待试玩锚点；每枚碎片优先选择不同的可见敌人，命中一次后消失。碎片可以击退但不应连续分裂。黑冰是短寿命墙面区域，只由冰霜弹珠撞墙留下，且每块墙面只保留一小段可辨认的冰层。

## 反馈与可读性

冻结、可推动冰球、即将碎裂三者必须不同：普通冻结偏静止，永冻冰球有更粗的外壳与地面投影，冰爆选中的目标有短促的裂纹聚焦。冰碎片的数量、路径和命中顺序应可跟踪；碰撞伤害要有方向性，帮助玩家理解是自己摆出的冰球位置造成了收益。

## 平衡边界

冰应奖励空间操作，不能用永久控制压制敌人。冻结必须有可预期的恢复或本发结束边界；永冻的同场冰球数量需要上限，首轮可试 2–3 个。冰爆以失去一个冰球和该目标全部霜冻交换碎片收益，不能绕过目标生命。黑冰只保留速度或回响收益，不应同时提供大额伤害、无限反弹与永久墙面。

## 与其他流派的桥梁

热冲击 `thermal_shock` 使燃烧和冻结共存的目标碎裂并解除两态，连接火。黑冰 `black_ice` 将冰霜弹珠的撞墙转为自铺冰面，使回响弹珠在其上反弹不掉速。冻脆 `brittle` 让冻结敌人受到爆炸或击飞时额外碎裂，连接炸弹。冰与闪电的聚团收益属于自然涌现，不额外承诺专属桥梁。

## 待确认问题

1. 冰球的推行速度、重量感与转向阻力怎样才能既可控又不显迟滞？
2. 永冻冰球的数量上限应按全场还是按单次发射约束？
3. 冰爆碎片应偏向自动选敌还是允许玩家通过发射方向影响优先目标？
4. 黑冰的可见范围和寿命怎样设置，才能支持回响而不让台面长期被蓝黑区域覆盖？
