# 遗物 / 流派构筑设计文档（v4 决策草案）

> 状态：v4 决策草案 · 2026-07-22 · 已吸收第三轮用户确认；暴击与护甲仍待共同定案
>
> 依据：调研 `docs/explore/relic_system_research_zh.md`；现状勘察 `docs/architecture/relic-marble-skill-audit.md`。
>
> **v4 相对 v3 的关键变更（回应第三轮反馈）：**
> 1. **撤回技能重设计**：Dash / Magic Missile 保持现状，不作为构筑轴，也不为其新增专属遗物；v3 的聚团、牵引、标记与处决提案全部作废。
> 2. **毒彻底脱离火的模板**：感染改为**阶段制资源**（潜伏→发病→宿主），成熟主要**改变敌人行为**（减速/吸引/留孢子区/产生传播载体），payoff 是"把敌人变成传播媒介"而非"层数换 burst"。火保留"高节奏挥发网络"。两者机制真正不同。
> 3. **删除无法自圆其说的桥梁**：移除 `kinetic_transfer`(动能→冰无因果)、`grounding`(接地应泄电，名实相反)、`static_field`(桥梁套桥梁)、`fuse`(寄生于延迟炸弹)。**不再承诺"任意两流派可混构"**，改为给出**明确桥梁图**（8 条可信边 + 1 条涌现边）。
> 4. **主球规则放开**：允许特定弹珠担任主球，主球身份可成为流派弹珠的显著特征。
> 5. **新增暴击流派**：不沿用简单随机增伤模板；先确认印象、关键词和全新 Buff，再定弹珠与遗物清单。
> 6. **敌人物理与经济方向确认**：敌人物理纳入中期路线；经济重平衡获准；重开不保留全部物品。
>
> **阅读约定：** 【事实】源码勘察；【静态判断】不依赖试玩的推理；【提案】新设计，数值为待试玩锚点。
>
> **本轮已确认：** 允许特定弹珠担任主球；重开不保留全部物品；新增暴击流派；敌人物理改造纳入中期路线；同意经济重平衡；Dash 与 Magic Missile 保持现状、不重做。

## 0. 一句话结论

七个目标流派：火=高节奏挥发网络（涂满+引爆）、毒=延迟宿主经营（阶段成熟+把敌人变传染媒介）、冰=物理改造战场（敌人→冰球棋子）、回响=球体进化（反弹让球变形）、炸弹=保龄球位移引擎（炸成弹珠互撞）、闪电=导电网络（聚团放电）、暴击=待确认的精确爆发流派。Dash 与 Magic Missile **保持现状**，不承担流派拼图职责。桥梁只保留物理因果可信的少数边。

> **三问直答：** 联动偏弱【事实】；数值尚不能静态证明平衡但有明显风险【静态判断】；当前提案 = 6 个已展开流派 + 1 个待定案的暴击流派 + 23 件已列新遗物 + 暴击内容待补。技能不在本轮重做范围。

## 1. 调研要点回顾（详见 research）

- 强协同 = 动词链；但**每流派须有独特机制身份**，非同模板换数值。
- 桥梁须**主题可信**（机制能解释"为什么 A 影响 B"）；**不必覆盖所有两两组合**。
- 三种联动强度中，**机制改写**（改物理规则/资源用途/敌人行为）最造独特手感。
- 稀有度塑造节奏；弹珠肉鸽卖点在**物理改造**（板面空间即构筑空间）。
- 反模式：同质化、桥梁牵强、寄生窄遗物、无限递归、乘区不透明、物理随机压过决策。

## 3. Phase 0：基础设施（三切片，0a 已收窄）

### 3.1 切片 0a：统一伤害结算 + 稳定事件语义（行为不变迁移，最高优先）

**验收**：现有 4 遗物 + 6 弹珠 + 2 技能迁移后行为不变（GUT 回归）。**本切片不引入敌人物理/碰撞/击飞**——那些是流派原型基础设施（§5/§9）。

```gdscript
class_name DamagePacket
# source: marble_head | chain_segment | bomb | dot | relic_* | skill_dash | skill_missile | payoff_*
# element: physical|fire|poison|frost|lightning|arcane
# base / flat / is_dot / is_skill / is_relic / is_crit_eligible
# proc_coefficient (默认1.0) / generation (默认0)
# target: Enemy
```

**核心事件（仅稳定语义，不含物理系统）：**

| 钩子 | 时机 |
|---|---|
| `on_enemy_hit_by_marble(enemy, packet)` ★现有(带packet) | Head 命中 |
| `on_enemy_hit_resolved(enemy, was_burning, was_frozen, packet)` ★现有 | 命中结算后 |
| `on_poison_tick(enemy, stage/layers)` ★现有(带状态) | 毒 tick |
| `on_damage_dealt(enemy, packet, final)` 🆕 | 已纳入统一事件边界的伤害包结算后 |
| `on_burn_tick(enemy, layers)` 🆕 | 燃烧 tick |
| `on_status_applied(enemy, status, amount, packet)` 🆕 | 施加状态 |
| `on_enemy_defeated(enemy, packet)` 🆕 | 符合既定来源边界的击杀；技能来源待按现状核对 |

> **技能事件边界**：Dash / Missile 迁入统一伤害结算时，必须保持其当前可观察行为；是否参与通用击杀/伤害事件须在迁移前按现状另行核对，不能在流派设计中预设。无遗物以“技能”为专属触发键，任何流派也不得依赖技能命中或技能击杀才能成立。火葬等**弹珠命中专属**机制明确不被技能触发。

**事件顺序契约**：
```
on_enemy_hit_by_marble → 结算 packet → on_damage_dealt → 派生(on_status_applied) → 致死: on_enemy_defeated
```
零伤害包仍触发 on_hit（供计数）但 on_damage_dealt 带 final=0 不触发"造成伤害"类 payoff；衍生事件带 proc+generation，Effect 支持 cannotTriggerSelf。

> 物理类事件（`on_explosion` / `on_surface_bounce` / 敌人碰撞/位移）**不在 0a**，随各流派原型基础设施引入（§5、§9）。

### 3.2 切片 0b：状态模型（燃烧层数 + 感染阶段 + Frost 修复）

**验收**：燃料层数、感染阶段、Frost 时长、风箱时序 GUT 通过。

**燃烧 = 燃料层数**（火用，快速挥发）：命中+1 层，上限 10，叠层刷新时长，tick=层数×per_layer_dps，可传播/可引爆消耗。

**感染 = 阶段制**（毒用，延迟控制，**非层数 DPS**）：

| 阶段 | 行为 |
|---|---|
| Ⅰ 潜伏 | 已接种，轻微 tick；可被"潜伏期"件加速成熟 |
| Ⅱ 发病 | tick 增强；敌人**减速**（行为改变） |
| Ⅲ 宿主 | 成熟；敌人**被吸引向其他敌人**（移动传染源）；死亡→传播（孢子区/孢子体） |

- 修复 Frost 时长钳制；明确风箱时序（结算后判定或 UI 预告）。

### 3.3 切片 0c：内容投放与注册

弃用 EffectType 枚举作遗物注册键，改 `item.id→effect 脚本`；`Item` 增 `rarity/tags/weight/requires_tags`；商店接 RunRandomSource；补齐渠道；稀有度权重按 12 层/5 槽节奏反推（每局 Common ~8–10 / Uncommon ~3–4 / Rare ~1–2）。桥梁件用 requires_tags 资格过滤防死选。

## 4. 构筑解剖：5 遗物 / ≤3 弹珠 / 1 辅助技能

### 4.1 容量与构筑形态

| 槽位 | 容量 | 含义 |
|---|---|---|
| 弹珠 | ≤3 | 可上 2 种元素（主+副+通用）；**允许特定弹珠担任主球**，主球不再固定为暗影球 |
| 遗物 | 5 | 满配 1 流派（enable+2support+payoff+bridge），或主流(3)+副流 enable+bridge(2)=**双流派混构** |
| 技能 | 1 | 辅助工具（§4.4），按构筑选 Dash/Missile，不是流派 |

### 4.2 流派上线门槛

雏形=1 对应弹珠+1 核心遗物；成型=+2–3 件；混构=第 4–5 槽引入副流派或通用件。

### 4.3 流派总览（机制身份 + 诚实分级）

| 流派 | **独特机制身份** | 关键件 | 等级 |
|---|---|---|---|
| A 纵火 | 高节奏**挥发网络**：火星物理传播，主动引爆高燃料节点 | 风箱/火葬 | 🟡 |
| B 瘟疫 | 延迟**宿主经营**：阶段成熟改变敌人行为，死亡变传染媒介 | 潜伏/脓爆 | 🟡 |
| C 冰封 | **物理改造战场**：敌人→可推动冰球/障碍，靠碰撞与碎裂造伤（非DoT） | 永冻/冰爆 | 🔴 |
| D 回响 | **球体进化**：反弹充能使球变形（穿透/留痕/巨大化） | 共振核心/雪崩 | 🔴 |
| E 爆破 | **保龄球位移引擎**：把敌人炸成弹珠互撞（需敌人物理） | 震荡波/殉爆 | 🔴 |
| F 闪电 | **导电网络**：命中带电，靠近的带电敌人持续放电（需网络服务） | 极化/雷暴 | 🔴 |
| G 暴击（待定案） | 推荐方向：**可规划的精确爆发**，通过全新 Buff 蓄势并确定性兑现，而非纯随机概率 | 校准 Buff / 暴击弹珠与遗物待确认 | 🟡? |

> 等级：🟡 仅需状态模型+稳定事件（A/B，建在 0b 上）；🔴 需物理/系统级原型（C 冰球物理 / D 反弹钩子+球变形 / E 敌人物理碰撞 / F 导电网络服务）；暴击等级须在 Buff 语义确认后重评；⛔ Multiball 引擎级。

### 4.4 技能边界（已确认不改）

Dash 与 Magic Missile **保持当前设计和行为**。本轮不重做技能，不新增技能专属遗物，也不把技能视为任一流派的核心、供给件或 payoff。v3 曾提出的 Dash 聚团/轨迹修正与 Missile 标记/牵引/处决方案均已撤回，后续流派文档不得依赖这些不存在的能力。

## 7. 数值框架

### 7.1 伤害包模型（burst/传染/碎片走子包，不入全局乘区）

`packet_base=base+flat` → 按 source/element/is_dot 与 target 状态筛选适用倍率（命中倍率区 × 易伤区 × 暴击区）→ `round(...)` → 护甲结算（语义待确认）→ **火葬/孢子区/冰碎片/殉爆生成子 DamagePacket**（带 gen/proc）回到开头。击飞/碎片引入后携带 knockback，触发碰撞伤害（属流派原型，非 0a）。

### 7.2 护甲接入主线前的设计决策（待讨论）

【事实】当前伤害路径已有“每次伤害减去目标护甲、最低为 0”的规则与护甲/穿甲数值入口，但正常内容尚未形成稳定的护甲来源、敌人分布、可视反馈和构筑关系，因此还不能视为完整主线机制。

| 方案 | 玩家感受 | 对七流派的影响 | 判断 |
|---|---|---|---|
| A 每包固定减伤（现状语义） | 直观但容易频繁打出 0 | 强烈压制火/毒/闪电等小额多段，天然偏爱暴击/爆破的大击 | 不建议直接作为主线终版 |
| B 百分比减伤 + 护甲削减 | 各伤害频率相对公平，穿甲价值稳定 | 腐蚀可降减伤率，暴击不会因固定门槛获得隐性双重优势 | **推荐作为讨论起点** |
| C 可击破的护甲条 | 先破甲、后爆发，节奏最鲜明 | 可形成独立“破甲→处决”循环，但容易像第二条生命，需更多反馈与内容 | 可作为精英/Boss 特性，不宜先全局铺开 |

待确认：护甲属于敌人的常驻属性还是少数敌人特性；DOT/溅射/碰撞是否同等受护甲影响；穿甲是降低护甲值、忽略比例，还是只服务物理伤害；`corrosion` 腐蚀是否继续作为毒↔物理桥梁。

### 7.3 现有数值静态判断【静态判断】

不能静态证明平衡（缺命中频率/每发时长/AOE目标数/护甲分布），但风险明显：开局100金购买力过高；精英双奖励 vs 有限遗物槽→后期经济失约束；Boss 固定240HP 被乘区碾压；敌人只+5HP/层纯线性。
**首轮锚点（待试玩）**：弱敌15HP~5–8命中；强敌40HP~12–18命中或靠状态；enable 级~+30–50%该轴；payoff 级成型~×1.5–2；AOE 按2–3目标；每局候选~6–8件、升级3–4件、成型1主流+1副流（5槽）。

### 7.4 叠加/阈值/滚雪球

燃料/感染设上限或阶段封顶；阈值 HUD 可见；成长件需 tempo cost（闷烧要时间、感染要等待、雪崩要轨迹）；容量闸门（槽/每发碰撞计数/gen/proc）；章节提升敌人机制复杂度而非只抬 HP。
