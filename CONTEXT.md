# Pinball Rogue 领域语言

本文件统一玩家构筑、关卡与商店规则中的领域术语，避免同一概念在设计和实现中出现不同含义。

## Language

**同一物品**：
遗物和技能拥有相同的唯一身份，或弹珠属于同一弹珠类型时，视为同一物品；同一物品共享等级且不能作为独立副本重复持有。
_避免使用_：相同资源、同一实例

**技能等级**：
当前装备技能的成长阶段；技能被替换时，其等级进度随旧技能一并失去，再次获得该技能时按新报价等级开始。

**升级报价**：
商店针对玩家已拥有且仍可升级的同一遗物、弹珠或技能给出的商品报价；购买后提升现有物品，不产生第二份副本。
_避免使用_：重复商品、重复物品

**状态修饰（Buff）**：
宿主（敌人）身上带持续时间与叠层的运行时状态，由 `BuffDef` 定义、`BuffHost` 承载，定义来源唯一为 `BuffRegistry`；毒、冰霜、冻结、燃烧等战斗减益均属此类。Buff 只通过 typed 事件（`buff_ticked`）对外暴露离散 tick，不反向调用 Effect 域。
_避免使用_：状态效果、全局 Buff（全局属性修正服务已删除）

**遗物效果（Effect）**：
由玩家持有的遗物驱动、按战斗事件（命中、命中结算、毒 tick）触发的 proc 行为（闪电链、风箱、毒培养、冰锤），由 `EffectManager` 从唯一脚本表 `EffectRegistry` 同步实例。依赖方向单向：Effect 可经宿主门面（`add_buff` 等）施加 Buff，Buff 绝不反向调用 Effect。
_避免使用_：遗物技能、遗物 Buff
